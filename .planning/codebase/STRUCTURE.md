## Directory Purposes

**Rokt.Widget/src:**

- Purpose: Main SDK source code - all TypeScript interfaces, components, and type definitions
- Contains: TypeScript files (.ts, .tsx) implementing the public API
- Key files: `Rokt.tsx`, `index.tsx`, `rokt-embedded-view.ts`
- Pattern: Flat structure with platform-specific files using `.ios.tsx` and `.android.tsx` suffixes
- Build output: TypeScript compiled to JavaScript in `dist/` directory

**Rokt.Widget/android:**

- Purpose: Android native implementation of bridge between React Native and Rokt Android SDK
- Contains: Kotlin and Java source files organized by architecture type
- Structure:
  - `src/main/java/com/rokt/reactnativesdk/` - Shared implementation (RNRoktWidgetModuleImpl, RoktEmbeddedViewPackage)
  - `src/newarch/java/com/rokt/reactnativesdk/` - New Architecture TurboModule implementations
  - `src/oldarch/java/com/rokt/reactnativesdk/` - Old Architecture (pre-TurboModule) implementations
  - `src/main/AndroidManifest.xml` - Android package declaration
- Build config: `build.gradle`, `gradle.properties`
- Key classes:
  - `RNRoktWidgetModuleImpl`: Core module logic for all callbacks and state management
  - `RNRoktWidgetModule` (newarch): TurboModule wrapper
  - `RoktEmbeddedViewManager`: Native UI component manager for embedded placements

**Rokt.Widget/ios:**

- Purpose: iOS native implementation of bridge between React Native and Rokt iOS SDK
- Contains: Objective-C/C++ (.m, .h, .mm) files
- Key files:
  - `RNRoktWidget.h` / `RNRoktWidget.mm` - Main bridge module (supports both architectures)
  - `RoktEventManager.h` / `RoktEventManager.m` - Event emission to JavaScript
  - `RoktWidgetManager.m` - Native component manager
  - `RoktNativeWidgetComponentView.h` / `.mm` - New Architecture Fabric component
- Xcode project: `RNRoktWidget.xcodeproj`

**Rokt.Widget/plugin:**

- Purpose: Expo config plugin for automatic native linking in Expo projects
- Contains: TypeScript implementation of Expo ConfigPlugin interface
- Location: `src/withRoktSDK.ts`
- Purpose: Currently a placeholder; autolinking handles most configuration

**RoktSampleApp/src:**

- Purpose: Reference implementation showing how to use the SDK
- Contains: React Native components demonstrating SDK integration
- Key file: `App.tsx` - Main component showing:
  - SDK initialization
  - Placement execution with custom attributes
  - Event listening and handling
  - Two-step fulfillment flow
- Utils: `utils/rokt-constants.ts` - Configuration constants for testing

**RoktSampleApp/android & ios:**

- Purpose: Native project scaffolding for sample app
- Contains: Platform-specific app configuration and resources
- iOS: Xcode project structure with CocoaPods integration
- Android: Gradle-based build configuration with SDK dependencies

**ExpoTestApp:**

- Purpose: Test integration in Expo managed workflow environment
- Structure: Minimal Expo application with SDK dependency

## Naming Conventions

**Files:**

- **TypeScript source**: camelCase (e.g., `rokt-embedded-view.ts`, `RoktNativeWidgetNativeComponent.ts`)
- **Platform-specific**: `<name>.ios.tsx` and `<name>.android.tsx` for platform variants
- **Native Kotlin**: PascalCase classes, package name `com.rokt.reactnativesdk`
- **Native Objective-C**: Prefix with `RN` or `Rokt` for React Native modules (e.g., `RNRoktWidget.h`)
- **Compiled output**: TypeScript → JavaScript in `dist/` directory (same file structure)

**Directories:**

- **Source code**: `src/` for TypeScript, `android/src/` for Kotlin, `ios/` for Objective-C
- **Native architecture variants**: `android/src/newarch/` and `android/src/oldarch/` for React Native architecture versions
- **Compiled output**: `dist/` for built JavaScript
- **Build artifacts**: Generated files not committed to git

**Classes and Interfaces:**

- **Exports**: PascalCase with semantic naming (Rokt, RoktEmbeddedView, CacheConfig, RoktConfigBuilder)
- **Internal interfaces**: IBridge or Impl suffix for implementation details
- **React components**: PascalCase (RoktEmbeddedView)
- **Configuration types**: "Config" or "Builder" suffix (RoktConfigBuilder, CacheConfig)

**Methods:**

- **camelCase** for all functions and methods
- **Verb-first** for action methods: initialize, execute, setEnvironmentToStage, purchaseFinalized
- **Get/set** prefix for property accessors: getSessionId, setSessionId

## Where to Add New Code

**New Feature (Core SDK API):**

- Primary code: `Rokt.Widget/src/Rokt.tsx` - Add new static methods to Rokt class
- Native spec: `Rokt.Widget/src/NativeRoktWidget.ts` - Add method to TurboModule Spec interface
- Android: `Rokt.Widget/android/src/main/java/com/rokt/reactnativesdk/RNRoktWidgetModuleImpl.kt` - Implement method logic
- iOS: `Rokt.Widget/ios/RNRoktWidget.mm` - Implement native bridge
- Type definitions: Add to `Rokt.tsx` if new config types needed
- Sample usage: Add to `RoktSampleApp/App.tsx` to demonstrate

**New Component/Module:**

- Implementation: `Rokt.Widget/src/<new-component>.tsx`
- If platform-specific: `Rokt.Widget/src/<new-component>.ios.tsx` and `.android.tsx`
- Platform selector: `Rokt.Widget/src/<new-component>.ts` (imports both, exports based on Platform.OS)
- Export: Add to `Rokt.Widget/src/index.tsx`

**Configuration/Types:**

- New config options: Add to `IRoktConfig` interface in `src/Rokt.tsx`
- Builder method: Add to `RoktConfigBuilder` class in `src/Rokt.tsx`
- Native types: Define in `RNRoktWidgetModuleImpl.kt` (Android) and native code (iOS)

**Utilities:**

- Shared helpers: Consider if they belong here or in the sample app
- SDK utilities (type conversions, validators): `Rokt.Widget/src/` alongside main code
- Sample app utilities: `RoktSampleApp/utils/`

## Special Directories

**Rokt.Widget/dist:**

- Purpose: Compiled JavaScript output from TypeScript build
- Generated: Yes (via `npm run build` which runs `tsc`)
- Committed: No - included in `.gitignore`, generated during publish
- Content: Transpiled JS and `.d.ts` declaration files matching `src/` structure

**Rokt.Widget/plugin/build:**

- Purpose: Compiled Expo plugin code
- Generated: Yes (via `npm run build:plugin`)
- Committed: No
- Content: Plugin compiled JavaScript for Expo config injection

**Android Native Architecture Split:**

- **Why:** React Native transitioned from Old Architecture (ViewManager/Bridge) to New Architecture (TurboModule/Fabric)
- `android/src/oldarch/`: Compiled when `newArchEnabled=false`
- `android/src/newarch/`: Compiled when `newArchEnabled=true`
- `android/src/main/`: Shared code used by both (RNRoktWidgetModuleImpl, RoktEmbeddedViewPackage)
- Gradle build variants automatically select correct implementation

**iOS New Architecture Support:**

- Both implementations coexist in single files using conditional compilation
- `#ifdef RCT_NEW_ARCH_ENABLED` guards in `.h` and `.mm` files
- Runtime detection in `Rokt.tsx` lines 11-32 selects appropriate TurboModule

**Node_modules:**

- Purpose: Installed dependencies
- Generated: Yes (via `npm install`)
- Committed: No - included in `.gitignore`
- Key dependencies:
  - `react-native`: Main React Native framework
  - `typescript`: TypeScript compiler
  - `@expo/config-plugins`: Expo plugin framework
  - `@types/react-native`: Type definitions for React Native
